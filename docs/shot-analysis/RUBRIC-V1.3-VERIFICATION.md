## 1. 为什么没跑完:free tier 日配额(运营事实,首次确认)

正式 3×3 于 2026-07-09 下午多次中断,先 503(上游容量高峰)后持续 429。
探针直查 API 确认根因:**当前 GEMINI_API_KEY 是 free tier,
`generate_content_free_tier_requests` 限额 20 次/天(gemini-2.5-flash)**。
当天配额被复验重试与探针耗尽;下一个窗口:北京时间次日 15:00(Pacific 午夜)重置。

运营含义:
- 一次 3×3 协议 = 18 次请求,几乎吃满一天配额,当天不能再有其他分析调用;
- 此前成本文档中的美元数字是名义值(free tier 实际不计费),但**配额本身成了硬约束**;
- 若拉片要进入日常使用(每集 2 次请求),free tier 每天最多 ~10 集,需要考虑升级付费层。

## 3. 口径修订(KB v1.3.1,与 v1.2 测量边界同方法论)

graphics_overlay_dependency 操作化为「**物件 ≠ 内容**」:
- hit:description **明确要求呈现具体图形/界面内容**(UI 弹窗及其内容、地图/定位界面、
  字幕卡、片头动画、明确写出的屏幕显示内容);
- no_hit:仅出现电脑/手机/屏幕/图纸/名片/手表等**物件**而未写明呈现内容
  (物件本身可由生成链路产出);
- 画面文字可读性归 text_in_frame 单独衡量,两维度不再重复计同一风险。

新增 2 个真实库内 few-shot 反例(看笔记本电脑镜头、俯视图纸镜头)钉住边界。
预期效果:消除 11↔5 级别的裁决摆动;副作用:视频 1 的 postProduction 分数
预计稳定在 8(低命中口径),消费方应知晓这是口径收紧而非片子变了。
