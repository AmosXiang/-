# 工作流重设计方案 v2.1（2026-07-14 定稿，P0 已获准开工）

> v1 基于用户当日反馈起草；v2 吸收外部 review（Codex）修订；v2.1 吸收 Codex 第二轮意见：
> 抽屉边界收紧、分镜状态拆三维、导出双模式防静默漏页、P2 内部拆 P2a/P2b。
> 逐条决议见 §八。**核心增补：评审 / 定稿 / 版本机制（§四）**——v1 把创作理解成一次性向导，
> 而导演工作实际是反复生成、比较、修改、定稿的循环。

## 一、用户反馈的问题清单

1. **首次启动即展示演示样本**（Pro_Edit_Final_V4.mp4），首次用户没有影片，主页应当是"上传优先"的落地页；上传后才进入分析布局。上传应是独立页面。
2. **顶部 tab 结构混乱**：分镜脉络 / 人物画像 / 叙事与爽点是"分析"功能，创意生成是"创作"功能，不属于同一结构，创意生成应独立成页。
3. **视频分析记录**（当前在左栏）需要更合理的位置。
4. **创意工作流内部**（风格设定 → 角色配置 → 分镜生成 → 导出）：
   - 故事创意概览需要**可编辑 + 保存**，作为分镜生成的参考依据；
   - 分镜画面可编辑项不够，需要 ComfyUI 级高级调整，但要保证**各分镜画风统一**；
   - **视频生成先移除**：不与分镜生成叠加，当前视频生成也不好用，先把分镜生成打磨好；
   - 导出物改为 **PPT 式分镜手册**：说明每个分镜是什么、情节如何、之后如何用于视频生成。
5. **Bug**：
   - 分镜生成模块鼠标滚轮无法下滑，下方分镜看不到；
   - 点击某个分镜偶发黑屏，需刷新；
   - 刷新后回到最初始页面（状态全丢）。

## 二、代码 / 界面层面的根因确认

| 现象 | 根因 | 位置 |
| --- | --- | --- |
| 刷新丢状态、回到初始页 | 整个应用是单页 `useState`，**无路由、无持久化**；刷新即重置 | `src/App.tsx`（6881 行单组件） |
| 点分镜黑屏 | **无 ErrorBoundary**（`main.tsx` 直接渲染 `<App/>`），任一渲染异常（如旧数据缺 `camera`/`framing` 字段被直接解构）即白/黑屏 | `src/main.tsx`、App.tsx 各 shot 渲染路径 |

| 演示样本常驻 | 无记录选中时默认渲染 `videoAnalysisData` 演示数据 | `App.tsx:37`、`src/data.ts` |
| 角色三视图等**破图** | 资源 404 / 未生成时直接渲染 `<img>`，无失败态与占位态 | 创意概览角色卡等处 |

## 三、新信息架构（IA · v2）

### 全局骨架

顶栏左侧常驻**两个一级入口：素材库 ｜ 创意项目**；右侧为**系统状态区 + 抽屉**。
抽屉边界收紧（v2.1），**只放系统健康类内容**：ComfyUI 健康状态、当前任务队列、
失败任务与重试入口、后台运行记录。**不放**："导出 JSON 报告"归入对应项目的
「导出交付」步骤或项目更多菜单；"历史视频资产"归入对应创意项目的**「项目资产」**
（只读，可播放/下载），不藏在系统健康入口里——抽屉不能变成杂物箱。
分析工作台是素材库的下钻页，创意工作室是创意项目的下钻页；
用户在任何页面都能通过全局导航离开，"以此为模板创建项目"只是主要桥梁而非唯一出口。

```
#/library                 素材库（首页）
#/analysis/:videoId       拉片工作台（素材库下钻）
#/studio                  创意项目列表
#/studio/:projectId       创意工作室（创意项目下钻）
（未来）#/video-lab        视频实验室（本轮不做）
```

### 1. `#/library` 素材库 = 首页
- **无任何记录时**：整页居中的大上传区（拖拽/点击 + 竖屏短剧模式开关），下方一行"或试用演示样本"小入口（演示数据不再默认加载）。
- **有记录时**：上传区收窄置顶，下方是分析记录**卡片网格**（封面帧、标题、类型/标签、分镜数、日期；筛选/搜索沿用现有逻辑）。
- 点击卡片 → `#/analysis/:videoId`；上传并分析完成 → 自动跳转。

### 2. `#/analysis/:videoId` 拉片工作台
- tab 只留分析三件套：**分镜脉络 / 人物画像 / 叙事与爽点**；不同 tab 右侧展示对应上下文。
- 左栏为**当前视频的分镜时间轴导航**；右侧统计改为**可折叠摘要**，不永久占宽。
- 顶栏"切换视频"下拉（最近记录快速跳转）。
- 故事发展脉络 / 爽点条目**可点击跳转对应时间码与镜头**（P3）。
- 主 CTA：**「以此为模板 → 创建创意项目」** → `#/studio/new?template=:videoId`。

### 3. `#/studio/:projectId` 创意工作室
**步骤条保留在顶部**（水平、可回跳），不做左侧竖排——分镜阶段已有三列，再加竖栏会形成四列竞争：

```
① 故事编辑 → ② 风格设定 → ③ 角色配置 → ④ 分镜生成 → ⑤ 导出交付
```

- **① 故事编辑（新增）**：结构化分区编辑而非一整块文本框，v1 先映射现有数据模型
  （标题｜三幕结构=`structure`｜视听节奏=`rhythm`｜高潮与爽点=`climaxDesign`｜分镜大纲=`newShots` 描述），
  保存走已有 `PUT /api/generated-scripts/:id`；"关键转折 / 角色关系与人物目标"等新字段后置，
  避免编辑器 schema 与生成 prompt 脱节。
- **② 风格设定**：即"风格契约"锁定处（见 §六），批量生成前必须完成。
- **④ 分镜生成**：三列布局保留（分镜列表｜主画布｜检查器）；中央 tab 去掉「动态视频」。
  **检查器按五区重组**：
  1. 镜头意图与情节；
  2. 结构参数（构图、机位、景别、人物走位）；
  3. 项目级风格契约——**只读**展示锁定项；
  4. 当前分镜可调整项（seed 等）；
  5. 生成版本与历史（见 §四）。
  **机位派生**移入检查器"工具"区段按需展开（它是一次性工具操作，不该常驻顶部）；
  批量派生沿用 preflight 两态，确认前展示选中数量、预计生成张数与预计耗时。
- **⑤ 导出交付**：见 §五。
- **视频生成退出主流程**：移除的是"调用 Provider 生成视频"的操作入口；
  运镜/速度/时长等**导演信息保留**并进入分镜手册。已生成的 MP4 不失去入口——
  归入对应创意项目的**「项目资产」区**（只读列表 + 播放/下载，v2.1 从系统抽屉移出），
  与 Agnes 数据保留原则一致；后端 `/api/generate-video`、`/api/video-tasks` 等 API 保留，
  未来迁入 `#/video-lab`。
- 文案修正：机位派生面板"已保存到云端"→"已保存到本机"。

### 「模块终点」拍板

以**「分镜交付包」为分镜生成模块的正式终点产物**（理由：稳定中间格式、人机两用、
与视频生成解耦，现有视频后端资产未来由 Video Lab 复用）。

## 四、评审 · 定稿 · 版本机制（v2 新增，方案核心）

导演工作流是"反复生成 → 比较 → 修改 → 定稿"的循环，**"已生成"不等于"完成"**。

1. **状态拆三个正交维度**（v2.1，避免单一状态机的冲突——如"已定稿的镜头又在生成新版本"、
   "新版本失败不应拖累原定稿"、"定稿后故事修改应显示『已定稿·基于旧输入』而非丢失定稿"）：

   | 维度 | 状态 | 来源 |
   | --- | --- | --- |
   | 评审状态 | 未生成 / 待评审 / 已定稿 | `finalTaskId` 与版本存在性派生 |
   | 输入新鲜度 | 当前 / 基于旧输入 | `isStale` |
   | 当前任务 | 空闲 / 排队 / 生成中 / 失败 | comfyui_tasks 实时态派生 |

   UI 不需要三个复杂控件：列表主徽标显示评审状态（待评审/已定稿），
   旁边按需叠加「已过期」角标或任务进度/失败标记；不只靠颜色表达。
2. **版本体系**：每次生成产出一个版本。后端 `comfyui_tasks` 已按 shot 留存任务、prompt 快照与结果，
   版本历史**近乎零成本**；Shot JSON 增加 `imageStatus` / `finalTaskId`（延续 camera-derive 的
   可选字段模式，旧数据天然兼容）。前端支持同一分镜多版本**并排对比、设为定稿、回退**。
3. **上游变更提示（v1 简版）**：故事 / 风格 / 角色保存时，若存在已生成分镜，
   提示"N 个分镜基于旧输入生成"，用户可**批量标记「已过期」或选择保留**；
   判定依据 = 生成时的 prompt / 参数快照比对（`cameraPromptUsed` 与任务行 prompt 列已有先例）。
   全量依赖图追踪成本高，后置。
4. **导出默认只取「已定稿」版本**。

## 五、导出：分镜交付包（PPTX + manifest）

- **交付前检查**（导出页首屏）：已定稿 x/N、缺图 n、失败 n、缺参数 n；未达标项可点击跳回对应分镜。
- **双模式导出，不允许静默漏镜头**（v2.1）：
  - **正式交付包**（主按钮）：仅当全部镜头已定稿时可用；有缺项时按钮受阻并明确列出缺哪些镜头、缺什么；
  - **审阅稿**（次级按钮，随时可用）：保留完整镜头顺序，未定稿镜头生成占位页并标注 `DRAFT`，
    封面注明"审阅稿 · 已定稿 x/N"。
- 产物（两种模式同构）：
  1. `*.pptx` 分镜手册——封面页（项目标题、题材、风格契约摘要、角色表）+ 每分镜一页
     （定稿图、时间码/时长、运镜 move/speed、景别、机位 cameraH/V/zoom 及主帧派生来源、
     情节描述、视频生成参考提示词与建议 provider 参数）；
  2. `storyboard-manifest.json` 同构机器可读清单（未来 Video Lab 的输入）；
  3. 定稿图文件目录。
- 格式已拍板：**PPTX（不做 PDF）**。实现：`GET /api/generated-scripts/:id/export-deck`。

## 六、画风统一策略（风格契约）

核心思想：**风格是项目级契约，分镜只许动"结构"，不许动"风格"。**

1. **项目级「风格契约」**在 ② 风格设定锁定（复用现有 `artDirection.overlay` 与 per-script
   `comfyui-preferences`）：checkpoint / LoRA、风格 overlay 提示词、负面词、采样器 / steps / CFG / 分辨率。
2. **批量生成 preflight 强制校验**：契约未锁定则不允许批量生成（单张试跑可放行）。
3. **分镜级高级调整只暴露结构参数**：构图、机位（接 camera-derive 词表）、走位、seed；
   风格区块只读显示"由项目风格契约控制"。
4. **首图定稿 → 风格锚点**：定稿的首张分镜图可标记为风格参考，后续分镜经 IPAdapter/style-reference
   注入（P3，依赖 ComfyUI 工作流预设扩展）。
5. **同场景多机位**：走 camera-derive 主帧派生（已上线），天然同风格。
